# 应急调度系统操作说明文档

## 一、系统概述

应急调度系统是一个基于地理信息系统的综合应急资源调度与指挥平台，提供实时资源监控、协同标绘、空间分析和轨迹回放等核心功能。

### 技术架构
- **前端**：React 18 + TypeScript + Ant Design 5 + OpenLayers 9.0+
- **后端**：Node.js 18+ + Express.js + TypeScript + MySQL 8.0+
- **地图服务**：OpenLayers地图引擎 + GeoServer空间数据服务
- **路径规划**：GraphHopper/OSRM路径规划引擎
- **实时通信**：WebSocket双向通信

## 二、菜单模块功能说明

系统左侧菜单包含以下功能模块：

### 2.1 指挥大屏（Dashboard）
**功能说明**：系统主控中心，提供整体态势监控和核心数据展示

**主要功能**：
- 实时资源监控：显示资源总数、在线数量、可用数量
- 告警统计：展示当前告警总数、严重告警、待处理告警
- 应急事件监控：显示当前应急事件列表和状态
- 资源列表展示：支持列表模式和聚合模式切换
- 实时状态更新：通过WebSocket推送资源状态变化

**使用场景**：指挥人员查看系统整体运行状态，快速了解应急资源分布和告警情况

### 2.2 资源监控（Resource）
#### 2.2.1 实时监控
**功能说明**：实时监控所有应急资源的状态和位置信息

**主要功能**：
- 资源列表展示：显示所有注册的应急资源
- 实时状态更新：在线/离线/任务中状态实时显示
- 国际通用资源图标：根据资源类型显示不同的标准图标
  - 🚑 救护车：红色十字车辆图标
  - 🚒 消防车：橙色消防车辆图标
  - 🚓 警车：蓝色警车图标
  - 📡 传感器：绿色信号图标
  - 👤 人员：蓝色人形图标
- 地图定位：点击资源在地图上显示当前位置
- 资源筛选：按类型、状态筛选资源
- 资源详情：查看资源完整信息

**使用场景**：监控人员实时查看应急资源分布，快速定位可用资源

#### 2.2.2 资源管理
**功能说明**：应急资源的增删改查管理

**主要功能**：
- 资源添加：新增应急资源信息
- 资源编辑：修改资源基本信息
- 资源删除：删除不再使用的资源
- 资源分类：按类型管理资源（车辆、设备、人员等）
- 资源状态管理：手动调整资源状态

**使用场景**：管理员维护应急资源库，确保资源信息准确完整

### 2.3 事件管理（Incident）
#### 2.3.1 事件列表
**功能说明**：管理应急事件的全生命周期

**主要功能**：
- 事件创建：新建应急事件记录
- 事件查询：按状态、时间、类型筛选事件
- 事件详情：查看事件完整信息
- 事件状态管理：待处理/处理中/已完成状态切换
- 事件关联：关联相关资源和调度任务

**使用场景**：事件管理人员记录和跟踪应急事件处理过程

#### 2.3.2 事件地图
**功能说明**：在地图上可视化展示事件分布和状态

**主要功能**：
- 事件标记：在地图上标记事件位置（经纬度坐标）
- 事件状态显示：不同颜色表示不同状态（待处理-橙色、处理中-蓝色、已解决-绿色、已关闭-灰色）
- 事件等级显示：不同颜色表示不同等级（一般-绿色、重大-橙色、特大-红色）
- 国际通用事件图标：根据事件类型显示不同的标准图标
  - 🔥 火灾：红色火焰图标
  - ➕ 医疗急救：绿色十字图标
  - 🚗 交通事故：蓝色汽车图标
  - 🛡️ 公共安全：紫色盾牌图标
  - ⚠️ 自然灾害：橙色警告三角形图标
- 事件聚合：相邻的事件标记自动聚合显示，缩放地图可展开查看
- 事件详情查看：点击标记查看事件信息
- 事件筛选：按类型、等级、状态筛选事件
- 统计面板：实时显示各状态事件数量

**操作步骤**：
1. 进入"事件管理"菜单
2. 点击"事件地图"子菜单
3. 在筛选区域选择筛选条件（类型/等级/状态）
4. 地图上会显示符合条件的事件标记（聚合效果）
5. 滚动缩放地图可展开聚合查看单个事件
6. 点击事件标记查看事件详情

**使用场景**：指挥人员在地图上快速了解事件空间分布和影响范围

### 2.4 调度任务（Dispatch）
#### 2.4.1 任务管理
**功能说明**：创建和管理应急资源调度任务

**主要功能**：
- 任务创建：创建新的调度任务
- 任务分配：为任务分配应急资源
- 任务执行：跟踪任务执行进度
- 任务完成：标记任务完成状态
- 任务统计：统计任务执行情况

**使用场景**：调度人员根据应急事件创建调度任务，分配资源并跟踪执行

### 2.5 战术标绘（Plotting）
#### 2.5.1 标绘管理
**功能说明**：多用户协同在地图上进行战术标绘

**主要功能**：
- 智能路径吸附：绘制路径时自动吸附到道路网络
- 战术符号绘制：标记点、路径、区域绘制
- 实时协同：多用户实时同步标绘内容
- 图层管理：管理不同标绘图层
- 导出功能：导出标绘结果为GeoJSON格式

**使用场景**：多用户协同进行应急任务规划，在地图上标记重要位置和路线

### 2.6 轨迹回放（Playback）
#### 2.6.1 轨迹查询
**功能说明**：查询和回放应急资源历史轨迹

**主要功能**：
- 单车辆回放：查询并回放单个资源的历史轨迹
- 多车辆同步：多个资源轨迹同步回放
- 热力图分析：生成资源活动热力图
- 时间动态热力图：动态播放不同时间段的活动分布
- 统计数据：计算总里程、平均速度、最大速度等
- 导出功能：导出轨迹数据为CSV格式

**使用场景**：分析资源历史活动，评估调度效果，优化资源配置

### 2.7 空间分析（Spatial）
#### 2.7.1 等时圈分析
**功能说明**：分析从指定点出发在不同时间内可达的区域范围

**主要功能**：
- 等时圈生成：根据起点和时间生成等时圈
- 时间间隔设置：自定义等时圈时间间隔
- 交通方式选择：支持汽车、步行、自行车等
- 可达性评估：评估资源覆盖范围
- 结果导出：导出分析结果

**使用场景**：评估应急资源覆盖能力，优化资源部署位置

## 五、协同标绘与战术调度（TacticalPlotting）

### 5.1 功能概览
协同标绘模块支持多用户在地图上进行实时协同标绘，包括智能路径吸附、复杂战术符号绘制等功能。

### 5.2 智能路径吸附
智能路径吸附功能支持在绘制路径时自动吸附到道路网络，生成符合实际道路的可行驶路径。

**操作步骤**：
1. 进入"战术标绘"页面
2. 选择"路径"绘制模式
3. 在地图上点击添加路径起点
4. 继续点击添加途经点，系统自动吸附到最近道路
5. 双击或点击起点完成路径绘制
6. 点击"保存"按钮保存绘制结果

**技术说明**：
- 使用GraphHopper/OSRM路径规划引擎
- 自动计算最优行驶路径
- 支持避开限行区域和拥堵路段

### 5.3 复杂战术符号绘制
支持绘制多种战术符号，用于应急调度中的任务规划和区域标记。

**支持符号类型**：
- **标记点**：在地图上标记重要位置（如事故点、集结点）
- **路径**：绘制应急车辆行驶路线
- **区域**：划定作业范围、危险区域等

**操作步骤**：
1. 选择绘制模式（标记点/路径/区域）
2. 根据模式不同进行绘制：
   - **标记点**：点击地图添加标记点
   - **路径**：点击添加多个路径点，双击结束
   - **区域**：点击添加多个顶点，双击闭合区域
3. 点击"保存"按钮保存绘制内容
4. 支持删除已绘制的元素

### 5.4 协同功能
- 多用户实时同步：标绘内容通过WebSocket实时同步给所有在线用户
- 图层管理：支持显示/隐藏不同图层
- 导出功能：支持将标绘结果导出为GeoJSON格式

## 六、空间分析（SpatialAnalysis）

### 6.1 功能概览
空间分析模块提供多种空间分析功能，包括等时圈分析、缓冲区分析、空间查询、邻近分析、空间统计和轨迹插值等，用于评估应急资源覆盖能力、分析影响范围和优化调度决策。

### 6.2 等时圈分析
等时圈分析可以计算从指定起点出发，在不同时间内可达的区域范围。

**操作步骤**：
1. 进入"空间分析"页面
2. 选择分析类型：等时圈分析
3. 在地图上点击选择分析起点
4. 设置时间参数：
   - 时间间隔：每个等时圈的时间间隔（如5分钟、10分钟）
   - 最大时间：最大覆盖时间（如30分钟、60分钟）
5. 选择交通方式：
   - 汽车道路行驶
   - 步行
   - 自行车
6. 点击"执行分析"按钮
7. 等待分析结果，系统在地图上渲染等时圈

**分析结果展示**：
- 不同颜色区分不同时间覆盖区域
- 鼠标悬停显示区域详情
- 支持导出分析结果

**API接口**：
- 请求路径：`POST /api/v1/spatial/isochrone`
- 请求参数：
  - `longitude`: 起点经度
  - `latitude`: 起点纬度
  - `interval`: 时间间隔（分钟）
  - `maxTime`: 最大时间（分钟）
  - `mode`: 交通方式（car/walk/bicycle）
- 响应格式：GeoJSON FeatureCollection

### 6.3 缓冲区分析
缓冲区分析可以计算指定点、线、面要素周围的缓冲区域，用于评估影响范围。

**操作步骤**：
1. 选择分析类型：缓冲区分析
2. 选择分析对象：
   - 选择已绘制的点要素
   - 选择已绘制的线要素
   - 选择已绘制的面要素
3. 设置缓冲距离：
   - 输入缓冲距离（米）
4. 点击"执行分析"按钮
5. 系统在地图上渲染缓冲区范围

**分析结果展示**：
- 半透明区域表示缓冲范围
- 显示缓冲区面积
- 支持导出分析结果

**API接口**：
- 请求路径：`POST /api/v1/spatial/buffer`
- 请求参数：
  - `geometry`: 输入几何体（GeoJSON格式）
  - `distance`: 缓冲距离（米）
  - `unit`: 距离单位（默认为meters）
- 响应格式：GeoJSON FeatureCollection

### 6.4 空间查询
空间查询可以查询指定空间范围内满足条件的资源、事件等要素。

**操作步骤**：
1. 选择分析类型：空间查询
2. 绘制查询范围：
   - 绘制圆形区域
   - 绘制矩形区域
   - 绘制多边形区域
3. 选择查询对象：
   - 应急资源
   - 应急事件
   - 标绘图层
4. 点击"执行查询"按钮
5. 系统返回查询结果并在地图上高亮显示

**查询结果展示**：
- 地图上高亮显示查询结果
- 列表显示详细信息
- 支持导出查询结果

**API接口**：
- 请求路径：`POST /api/v1/spatial/query`
- 请求参数：
  - `geometry`: 查询范围（GeoJSON格式）
  - `type`: 查询对象类型（resource/incident/plotting）
- 响应格式：JSON数组

### 6.5 邻近分析
邻近分析可以查找指定位置周围的最近资源或事件。

**操作步骤**：
1. 选择分析类型：邻近分析
2. 在地图上点击选择查询点
3. 设置查询条件：
   - 查询对象类型：应急资源/应急事件
   - 查询数量：返回最近的N个结果
   - 最大距离：限制查询范围（米）
4. 点击"执行分析"按钮
5. 系统返回最近的结果并显示在地图上

**查询结果展示**：
- 地图上显示查询点和最近要素
- 连线显示距离
- 列表显示详细信息和距离

**API接口**：
- 请求路径：`POST /api/v1/spatial/nearest`
- 请求参数：
  - `longitude`: 查询点经度
  - `latitude`: 查询点纬度
  - `type`: 查询对象类型
  - `limit`: 返回数量
  - `maxDistance`: 最大距离（米）
- 响应格式：JSON数组

### 6.6 空间统计
空间统计可以对指定范围内的资源、事件进行统计分析。

**操作步骤**：
1. 选择分析类型：空间统计
2. 绘制统计范围
3. 选择统计对象：
   - 应急资源统计
   - 应急事件统计
4. 点击"执行统计"按钮
5. 系统返回统计结果

**统计结果展示**：
- 总数统计
- 按类型分组统计
- 按状态分组统计
- 密度统计
- 导出统计报告

**API接口**：
- 请求路径：`POST /api/v1/spatial/statistics`
- 请求参数：
  - `geometry`: 统计范围（GeoJSON格式）
  - `type`: 统计对象类型
- 响应格式：JSON统计对象

### 6.7 轨迹插值
轨迹插值可以对稀疏的轨迹点进行插值，生成连续平滑的轨迹。

**操作步骤**：
1. 选择分析类型：轨迹插值
2. 选择资源轨迹
3. 设置插值参数：
   - 插值间隔：插值点之间的时间间隔（秒）
   - 插值方法：线性插值/样条插值
4. 点击"执行插值"按钮
5. 系统生成插值后的轨迹

**插值结果展示**：
- 地图上显示原始轨迹和插值轨迹
- 对比插值前后轨迹点数量
- 导出插值结果

**API接口**：
- 请求路径：`POST /api/v1/spatial/interpolate`
- 请求参数：
  - `resourceId`: 资源ID
  - `startTime`: 开始时间
  - `endTime`: 结束时间
  - `interval`: 插值间隔（秒）
  - `method`: 插值方法（linear/spline）
- 响应格式：GeoJSON LineString

### 6.8 参数说明
- **等时圈参数**：
  - 时间间隔：等时圈之间的时间差，影响精度和性能
  - 最大时间：分析的最大覆盖时间，建议不超过120分钟
  - 交通方式：不同交通方式的速度和可达性不同
- **缓冲区参数**：
  - 缓冲距离：缓冲区的半径，单位为米
  - 距离单位：支持meters、kilometers、feet等
- **邻近分析参数**：
  - 查询数量：返回最近结果的数量
  - 最大距离：限制查询范围，单位为米
- **轨迹插值参数**：
  - 插值间隔：插值点之间的时间间隔，建议60-300秒
  - 插值方法：线性插值速度快，样条插值更平滑

## 七、轨迹回放（Playback）

### 7.1 功能概览
轨迹回放模块支持历史轨迹查询、回放、热力图分析和统计功能，支持单车辆和多车辆同步回放。

### 7.2 单车辆轨迹回放

**操作步骤**：
1. 进入"轨迹回放"页面
2. 选择"单车辆回放"模式
3. 设置查询条件：
   - 选择资源（应急车辆/人员）
   - 选择时间范围（起止时间）
4. 点击"查询轨迹"按钮
5. 查询成功后地图显示轨迹路线

#### 8.2.4 WebSocket实时推送

## 十一、常见问题

### 11.1 头像上传成功但未显示
**问题原因**：
- JWT token字段不匹配
- 文件路径配置错误

**解决方法**：
1. 检查后端JWT payload字段是否使用`userName`
2. 检查`backend/src/middlewares/upload.middleware.ts`中的`UPLOAD_DIR`配置是否为`../../uploads`
3. 清除浏览器缓存重新登录

### 11.2 后端服务启动失败
**问题原因**：
- 数据库连接失败
- 端口被占用

**解决方法**：
1. 检查MySQL服务是否启动
2. 检查数据库配置是否正确
3. 检查8000端口是否被占用
4. 查看后端日志获取详细错误信息

### 11.3 地图无法加载
**问题原因**：
- 地图服务未启动
- 网络问题

**解决方法**：
1. 检查GeoServer服务是否启动
2. 检查网络连接
3. 打开浏览器开发者工具查看控制台错误

### 11.4 WebSocket连接失败
**问题原因**：
- 后端WebSocket服务未启动
- 防火墙阻止

**解决方法**：
1. 检查后端服务是否正常运行
2. 检查防火墙设置
3. 确认WS_BASE_URL配置正确

## 十三、数据库设计

### 13.1 数据库概述
系统采用MySQL 8.0+作为主数据库，使用InnoDB存储引擎，支持事务和外键约束。所有表使用UUID作为主键，确保分布式环境下的唯一性。

### 13.2 核心业务表设计

#### 13.2.1 应急资源表（t_resource）
**表说明**：存储所有应急资源的基础信息和位置数据。

**表结构**：
```sql
CREATE TABLE t_resource (
    id VARCHAR(36) PRIMARY KEY COMMENT '资源ID（UUID）',
    resource_type VARCHAR(50) NOT NULL COMMENT '资源类型：vehicle/person/sensor/material',
    resource_name VARCHAR(100) NOT NULL COMMENT '资源名称',
    resource_status VARCHAR(20) NOT NULL COMMENT '状态：online/offline/alarm/processing',
    longitude DECIMAL(11, 8) NOT NULL COMMENT '经度',
    latitude DECIMAL(11, 8) NOT NULL COMMENT '纬度',
    properties JSON COMMENT '扩展属性',
    department_id VARCHAR(36) COMMENT '所属部门ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_location (longitude, latitude),
    INDEX idx_status (resource_status),
    INDEX idx_type (resource_type),
    SPATIAL INDEX idx_spatial (POINT(longitude, latitude))
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='应急资源表';
```

**字段说明**：
- `id`：资源唯一标识，UUID格式
- `resource_type`：资源类型，vehicle（车辆）、person（人员）、sensor（传感器）、material（物资）
- `resource_status`：资源状态，online（在线）、offline（离线）、alarm（告警）、processing（任务中）
- `longitude/latitude`：资源当前位置，WGS84坐标系
- `properties`：JSON格式的扩展属性，存储资源特定信息
- `department_id`：所属部门ID，外键关联部门表

**扩展属性示例**：
```json
{
  "vehicle": {
    "vehicleType": "救护车",
    "plateNumber": "京A12345",
    "capacity": 4
  },
  "person": {
    "name": "张三",
    "position": "医生",
    "skills": ["急救", "心肺复苏"]
  }
}
```

### 13.4 数据库优化策略

- **分页查询**：大数据集采用分页查询，避免全表扫描

#### 13.4.3 数据归档
- **历史数据归档**：定期将历史轨迹数据归档到归档表

- **通知清理**：定期清理已读且超过保留期的通知

### 13.5 数据库备份与恢复

#### 13.5.1 备份策略
```bash
# 全量备份
mysqldump -u root -p emergency_dispatch > backup_$(date +%Y%m%d).sql

# 增量备份（基于binlog）
mysqlbinlog --start-datetime="2026-02-27 00:00:00" \
           --stop-datetime="2026-02-27 23:59:59" \
           mysql-bin.000001 > incremental_backup.sql
```

#### 13.5.2 恢复操作
```bash
# 恢复全量备份
mysql -u root -p emergency_dispatch < backup_20260227.sql

# 恢复增量备份
mysql -u root -p emergency_dispatch < incremental_backup.sql
```

## 十四、API接口说明

### 14.1 接口概述
系统提供RESTful API接口，支持应急资源管理、事件管理、空间分析等功能。所有接口使用JWT认证。

### 14.2 通用规范
- **基础URL**：`http://localhost:8000/api/v1`
- **认证方式**：Bearer Token（JWT）
- **请求格式**：JSON
- **响应格式**：JSON
- **字符编码**：UTF-8

### 14.3 认证接口

#### 14.3.1 用户登录
- **请求路径**：`POST /auth/login`
- **请求参数**：
  ```json
  {
    "username": "admin",
    "password": "admin123"
  }
  ```
- **响应示例**：
  ```json
  {
    "code": 200,
    "message": "登录成功",
    "data": {
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": "user-uuid",
        "userName": "admin",
        "role": "admin"
      }
    }
  }
  ```

#### 14.3.2 获取当前用户信息
- **请求路径**：`GET /auth/user`
- **请求头**：`Authorization: Bearer {token}`
- **响应示例**：
  ```json
  {
    "code": 200,
    "message": "获取成功",
    "data": {
      "id": "user-uuid",
      "userName": "admin",
      "role": "admin",
      "avatar": "/uploads/avatar.jpg"
    }
  }
  ```

### 14.4 资源管理接口

#### 14.4.1 获取资源列表
- **请求路径**：`GET /resources`
- **请求参数**：
  - `page`: 页码
  - `pageSize`: 每页数量
  - `type`: 资源类型（可选）
  - `status`: 资源状态（可选）
- **响应示例**：
  ```json
  {
    "code": 200,
    "message": "获取成功",
    "data": {
      "list": [...],
      "total": 100,
      "page": 1,
      "pageSize": 20
    }
  }
  ```

#### 14.4.2 获取资源详情
- **请求路径**：`GET /resources/:id`
- **响应示例**：
  ```json
  {
    "code": 200,
    "message": "获取成功",
    "data": {
      "id": "resource-uuid",
      "resourceType": "vehicle",
      "resourceName": "救护车1号",
      "resourceStatus": "online",
      "longitude": 116.404,
      "latitude": 39.915,
      "properties": {...}
    }
  }
  ```

#### 14.4.3 创建资源
- **请求路径**：`POST /resources`
- **请求参数**：
  ```json
  {
    "resourceType": "vehicle",
    "resourceName": "救护车1号",
    "longitude": 116.404,
    "latitude": 39.915,
    "properties": {
      "vehicleType": "救护车",
      "plateNumber": "京A12345",
      "capacity": 4
    }
  }
  ```

#### 14.4.4 更新资源
- **请求路径**：`PUT /resources/:id`
- **请求参数**：同创建资源

#### 14.4.5 删除资源
- **请求路径**：`DELETE /resources/:id`

### 14.5 事件管理接口

#### 14.5.1 获取事件列表
- **请求路径**：`GET /incidents`
- **请求参数**：
  - `page`: 页码
  - `pageSize`: 每页数量
  - `status`: 事件状态（可选）
  - `level`: 事件级别（可选）

#### 14.5.2 创建事件
- **请求路径**：`POST /incidents`
- **请求参数**：
  ```json
  {
    "incidentType": "交通事故",
    "incidentName": "XX路口交通事故",
    "incidentLevel": "high",
    "longitude": 116.404,
    "latitude": 39.915,
    "description": "事故描述..."
  }
  ```

### 14.6 空间分析接口

#### 14.6.1 等时圈分析
- **请求路径**：`POST /spatial/isochrone`
- **请求参数**：
  ```json
  {

    "latitude": 39.915,
    "interval": 5,
    "maxTime": 30,
  ```

#### 14.6.2 缓冲区分析
- **请求路径**：`POST /spatial/buffer`
- **请求参数**：
  ```json
  {
    "geometry": {

    },
    "distance": 1000,
  ```

#### 14.6.3 空间查询
- **请求路径**：`POST /spatial/query`
- **请求参数**：
  ```json

    "geometry": {

      "coordinates": [[...]]

    "type": "resource"
  }
  ```

- **请求路径**：`POST /spatial/nearest`
- **请求参数**：
  ```json
  {

    "latitude": 39.915,
    "type": "resource",
    "limit": 5,

  }
  ```

### 14.7 标绘管理接口

#### 14.7.1 获取标绘图层
- **请求路径**：`GET /plotting`

- **请求路径**：`POST /plotting`
