需求文档：城市智慧应急协同调度平台
1. 项目概述
本系统旨在建立一个高集成、高性能的城市级应急指挥中心。系统需在二维地图上实时集成全市范围内的应急资源（人员、车辆、物资、传感器），并提供复杂的空间分析能力，辅助指挥员在突发事件中进行资源动态调度、路径规划与态势复盘。
2. 核心功能模块
2.1 高性能全域资源监控 (Rendering & Scalability)
海量点位渲染： 支持在地图上同时加载并平滑操作超过 50,000 个 动态点位（如市政井盖、烟感、警务人员）。
技术要求： 必须使用 ol/layer/WebGLPoints 或自定义 WebGL 渲染器，避免传统 Canvas 渲染导致的 UI 阻塞。
多模态聚合 (Advanced Clustering)：
在低缩放层级下展示资源聚合点，聚合点需以 Canvas 动态饼图 形式展现各类资源（如：救护车占30%，消防车占70%）的构成比例。
支持聚合点的平滑拆解动画。
状态实时同步： 接入 WebSocket 流，点位需根据后端推送的状态（如：报警、离线、处理中）实时变更颜色和呼吸灯特效。
2.2 动态态势空间分析 (Spatial Intelligence)
多重等时圈 (Isochrone) 分析：
用户点击事故点，系统需基于路网拓扑实时生成 5/10/15 分钟救援到达圈。
复杂度： 需结合路径规划服务返回的 Polygon 进行展示，并利用 Turf.js 实时计算该区域内覆盖的敏感建筑（学校、医院）数量。
动态缓冲区 (Buffer) 与过滤：
支持根据灾害等级（如化学泄漏半径）生成多层环形缓冲区。
空间检索： 自动筛选并高亮落在缓冲区内的应急物资点，并弹出资源统计列表。
自由套索查询 (Lasso Query)：
支持用户手绘任意闭合曲线。系统需利用 R-Tree 空间索引 在毫秒级内返回选区内所有 Feature 的属性详情。
2.3 协同标绘与战术调度 (Tactical Plotting)
智能路径吸附 (Snapping & Routing)：
指挥员手绘应急路线时，线条必须自动吸附至 矢量切片 (MVT) 定义的道路中心线上。
支持拖拽路径中间点，动态重新请求 OSRM/GraphHopper 接口更新路径几何形状。
复杂战术符号：
实现非标准几何图形的绘制：如燕尾箭头（军事进攻）、钳形攻势、集结区（多边形边缘平滑化）。
交互要求： 图形需具备控制点，允许用户通过拖拽控制点实时重塑几何形状。
2.4 时空大数据回放 (Spatiotemporal Playback)
多车同步插值回放：
支持同时回放 100 辆应急车辆在过去 24 小时的行驶轨迹。
核心逻辑： 针对上报间隔不均匀的数据，前端需进行 线性或贝塞尔插值，确保回放时图标移动在 60FPS 下丝滑顺畅。
时序动态热力图：
展示全城报警密度的随时间演化。
技术难点： 热力图需支持时间轴联动，在渲染帧内根据 timestamp 动态更新权重，避免切片重载带来的闪烁。
3. 高级交互与视觉优化
3.1 视觉增强与地图滤镜
实时夜间模式： 在不更换底图图层的前提下，通过 prerender 事件劫持 Canvas 上下文，利用 Color Matrix (颜色矩阵) 将标准地图实时转换为暗色调（Dark Mode）。
地图卷帘 (Swipe)： 实现上下/左右滑动卷帘，用于对比事故现场的“灾前”与“灾后”高分辨卫星影像。
3.2 智能标注避让 (Decluttering)
标签排斥算法： 在高密度区域，标签需自动避让或隐藏，高优先级（如报警中）的标签强制显示。
动态引线： 当标签因避让被挤开时，需自动绘制一条连接标签与要素实际位置的动态引导线。
4. 技术挑战与性能指标 (Non-Functional Requirements)
4.1 坐标系深度兼容
多源纠偏： 系统需同时完美叠加 天地图 (CGCS2000)、高德 (GCJ-02) 及 内部CAD局部坐标系。
实时投影转换： 确保在地图旋转、缩放过程中，所有图层的偏移量控制在像素级范围内。
4.2 极限性能
OffscreenCanvas 应用： 将繁重的轨迹计算和粒子动画移交给 Web Worker 和离屏画布，确保主线程交互（平移、缩放）响应时间 < 100ms。
帧率指标： 在 10,000+ 动态要素同屏时，全局 FPS 不低于 50。
4.3 鲁棒性与导出
内存监控： 严格控制 ol/Feature 的创建与销毁，防止长时间运行导致的内存溢出。
4K 高清导出： 支持将当前所有 Canvas 图层（含 WebGL、标绘、滤镜）合并导出为 300DPI 的高清汇报图片。
5. 建议技术栈
引擎： OpenLayers 9.0+
计算： Turf.js, Proj4.js
后端配合： GeoServer / PostGIS / Node.js (WebSocket) / python / mysql
渲染辅助： WebGL, Canvas API, Web Worker
